# Contributing to CSS Uncenter

Welcome, agent of chaos. You've come to the right place. Here, `display: flex; justify-content: center; align-items: center;` is the enemy, and we don't negotiate with perfectly aligned layouts.

## Getting Started

```bash
git clone https://github.com/arnoldwender/css-uncenter.git
cd css-uncenter
npm install
npm run dev
```

Open [http://localhost:5173](http://localhost:5173). If everything looks perfectly centered, you haven't turned it on yet.

## Submitting a Pull Request

1. Fork the repo and create your branch from `main`
2. Make your changes
3. Make sure the app still builds (`npm run build`)
4. Open a PR explaining what you broke and why it's beautiful
5. Screenshots of before/after destruction are highly encouraged

## Code Style

- **TypeScript** — our chaos is typed, thank you very much
- **React** functional components with hooks
- The app itself should be well-written and well-structured. The *output* should be chaotic. Don't confuse the two.
- Physics calculations should feel believable. Elements drift, they don't teleport (unless that's the mode).

## Bug Reports

Open an issue with:
- What you expected to happen
- What actually happened
- Your browser and OS
- Whether the bug made things more or less centered (this matters)

## Feature Requests

New un-centering modes, chaos algorithms, or destruction metrics? Open an issue! If your idea makes a designer wince, it's probably perfect.

## The Golden Rule

Write clean code that produces messy layouts. Never the other way around.
